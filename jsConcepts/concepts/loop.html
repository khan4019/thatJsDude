Dont exectute length stupidly

The first statement is executed only once, before starting of the loop. This could save you calculating length againg and again. You can have multiple declaration or you can miss it completely.


//array length is calculated every single iteration
for(var i =0; i< myArr.length;; i++){
  //my awesome looping
}

//array length is calcaulted only once
for(var i =0, len = myArr.length; i<len; i++){
  //my awesome looping
}

//even you can hoist first statement
var i =0, 
    len = myArr.length;
for(; i<len; i++){
  //my awesome looping
}

for..in

for..in only iterate over all enumerable properties of the object itself and those the object inherits from its constructor's prototype. if you want to exclude inherited properties you can use hasOwnProperty

It is not guranteed that for..in will maintain an order. So, for..in should not be used to iterate over an Array where index order is important.


var obj = {a:1, b:2}
for(var i in obj){
  console.log(obj[i]);
