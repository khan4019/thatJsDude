1. Best way to Create Array

using constructor:

One of the ways to create array is to call the new Array() constructor.


var myArr = new Array();
myArr[0] = 'Happy';
myArr[1] = 'New';
myArr[2] = 'Year';

> myArr;
  = ["Happy", "New", "Year"]

> myArr.length;
  = 4

Additionally, you can set array length inside constructor. However, this is not recommended if you are not sure you will use the whole array. As this will create an array with "undefined" elements. Besides, you can pass elements.


var myArr = new Array(100);
> myArr;
  = [undefined × 100]

var myArr2 = new Array(2,3,4,5);
> myArr;  
  = [2, 3, 4, 5]
better alternative:

rather construct it as empty array. if you know intial elements, its recommended to put them there rather inserting them later.


var myArr = [];

//create with elements if you know
var myArr2 = ['You', 'Are', 'the', 'Best' ];

2. be careful to set array length

array.length if you only want to read it


var myArr = ['javaScript', 'is', 'my', 'favorite', 'language' ];
> myArr.length;
  = 5
Don't blindly set length:

if you set the length you resize the array. If you set length to a number higher than the actual length, insert "underfined" elements. On the otherhand, if you set length lower than the actual length, it removes all the elements after the newly set length. So, be careful to set array.length


myArr.length = 8;
> myArr;
  = ["javaScript", "is", "my", "favorite", "language", undefined × 3] //three undefined elements inserted

myArr.length = 4;
> myArr;
  = ["javaScript", "is", "my", "favorite"] //removed 4 elements

3. Don't randomly insert by Index

if you insert elements in an arry by index. Please make sure you insert the element in the right place. If you just put an index much higher than the length of the array, JavaScript will put "underfined" elements in between and waste a lot of memory


var myArr = [];
myArr[0] = 1;
myArr[1] = 2;
myArr[2] = 3;
myArr[3] = 4;

> myArr.length;
  = 4
//just enter an element in higher index
myArr[68] = 69;
> myArr.length;
  = 69
> myArr;
  =[1, 2, 3, 4, undefined × 64, 69]
Only use positive integer as Index:

Don't insert by negative (-ve) index. In fact, only use positive interger as index. Javascript will not give any error, they will simply be lost when you check array in bare eye. However they sticks, you can read as property or by using a for-in loop


var myArr = [1,2,3,4];
myArr[-7] = -7;
myArr[2.5] = 3.5;
myArr['myDude'] = 'thatJSDude';

> myArr.length;
  = 4
> myArr;
  = [1, 2, 3, 4]

While reading by index. you cant trust it. it doesnt throw OutOfIndex error. if you try to get elements by using index higher than the length or invalid index, it just returns "underfined"


var myArr = [1,2,3,4];

> myArr[2];
  = 3
> myArr[169];
  = undefined  
> myArr[2.5];
  = undefined
> myArr['myDude'];
  = undefined
> myArr[-7];
  = undefined
To soleve this problem:

If you have random index, use object/ Associated Array other than array. if you are looping, use for in loop


var myObj = {};
myObj[-7] = 7;
myObj[68] = 69;

> myObj;
  = Object {68: 69, -7: 7}
