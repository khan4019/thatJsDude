If you ask for a variable inside a function, JavaScript doesn't know right away whether this variable you are asking for is a global variable or local variable. So, it has to follow the following steps.
  • First (Local Scope): It checks local variable. if it is in the local scope will grab it and give it to you.
  • Second (Parameters): If the variable doesn't exists in the local scope then it check to the parameters. Similar to keys in the key holder (Image-17)
  • Third (Outer Scopes) If the variable you are asking for doesn't exists in the parameter then it starts walking outside of the function via scope chain of the current scope until it hits the end of the chain. The end of the scope is called "window" object in JavaScript.
  • Fourth (Window): If the variable exists in the global scope, it would be a property of the global object "window". JavaScript will grab it an give it to you.
  • Fifth (Undefined): If the variable doesn't exists in the window object, JavaScript will return "undefined". This means the variable you are looking for, doesn't exists.

Since local scope is the first place to look for variable, local variable gets priority over global.


var x = 9;

function getX(x){
  var x = 7;
  return x;  
}

getX(5); // 7

5. Access Global Variable

Climbing through the Scope Chain: If you are asking for a global variable from inside of a function, you are climbing through the scope chain. In the code example you are climbing the scope chain for 1,000,000. Every single time, you have to check localScope, parameters and then get outside of the function. If you have more than one function inside function, then it becomes even more expensive.


var myArr = [0,1,2,3, ...., 1000000];

function getSum(){
  var sum = 0;
  for (var i =0; i<1000000; i++){
    sum += myArr[i];
  }    
  return sum;
}

getSum();

If you have to access global variable for hundred times, you can have a local copy of the global variable. In that case you dont have to climb through the scope chain.


var myArr = [0,1,2,3, ...., 1000000];

function getSum(){
  var sum = 0;
  var localArr = myArr;
  for (var i =0; i<1000000; i++){
    sum += localArr[i];
  }    
  return sum;
}

getSum();

6. Hoisting

Hoist is a verb which means raise something by means of rope and pulley. If you have never seen a hoist in your life, look at Image-19. Hoist is used in the factories to raise heavy things. One benefit of raising something at the ceiling is, you can see it from any part of the room/factory. (Image-20)

JavaScript does the exact same thing to hoist (raise) variable declaration and function declaration at the top of function scope. so that those variables and functions could be used from any part of the function without raising no reference Exception.

Compare this with floating of stuff. or find good example

Image-19: Mechanical Hoist

Image-20: Hoisting Bike

Image-21:Hoisting Power

If you execute the following foo function, you will get undefined.


function foo(){
  return b;
  var b = 7;  
}

foo(); //undefined

While creating Context of a function, JavaScript execution engine, scan through the whole function and bring variable declaration at the top of the scope. Please note the value assignment will not be hoisted. Since, only variable declaration is hoisted, the default value of unassigned variable is "undefined". Hence, you will get "undefined" returned as value of "b".


function foo(){
  var b;
  return b;
  b = 7;
}
foo(); //undefined

Similarly if you assign a function to a variable (function expression) only the variable part will be hoisted. However if you have a function declaration, the full function will be hoisted.

Hoisting Process

You can google more about it or read spec or explore V8 code. And if you find any good materials, please let me know.

  • function: Any function declaration will be hoisted at the top.
  • variable: And then the variable declaration will be hoisted. If you have a function expression (set function to a variable), only the variable part will be hoisted. the assignment of the function to a variable will stay at its place.

In the following code block, you have a function expression where function is assigned to a variable "bar" and a function declaration named as "foo" and a variable "a".


function myFunc() {    
    var bar = function() {
            return 'world';
        };    
    function foo() {
        return 'fooed';
    }
    var a = 5;
}

The above mentioned code after hoisting will look like


function myFunc() {
    function foo() {
        return 'fooed';
    }    
    var bar;
    var a;    
    bar = function() {
            return 'world';
        };
    a = 5;        
}

Take Away

Only the declaration of a variable is hoisted. Value assigned remains at its location. Function declaration is hoisted. If you have a function expression only the variable part of the function expression will be hoisted.

7. Closure

if you have a function inside a function, execution of the inside function will create a scope inside of the outer function. Since, insider function scope is enclosed by outer function scope. The inner function scope is called closure. Remember, to be a closure, you dont have to return a function, you just have to call a function inside a function


function a(){
  function b(){
     console.log('closure');
  }
  b(); //creates closure
}

Useful Closure:: Closure is a powerful feature of JavaScript. Let's have a look into the following example. we have a outer function that has a local/private variable "i" and it return a function. Inside the returning function, the inner function returns the value of i and increases it by 1.


function counter(){
    var i = 0;
    
    return function(){
       return i++;
    }
}

Step-1: In the following core, we execute counter function and assign the result of counter to a variable "a". Hence, the value of a will be the inner function which would be return by executing function counter.

Step-2: Now if you execute function "a" (execute inner function), it will return 0 (value of i) and will increase it by 1. If you execute function "a" again it will return 1 (current value of i) and will increase the value of "i" by 1. Hence, you will get 2, 3, 4, 5 ....

This means, closure hold the state of local/private variable in the outer scope. And every time, you call it you get a value increased by 1.


var a = counter();
a(); //0
a(); //1
a(); //2
a(); //3
a(); //4
a(); //5

Now you can have another variable "b" and assign the execution result of the function counter. If you execute "b" (the inner function of counter), you will get 0. if you execute again, you will get 1. and then 2, 3, 4, ...


var b = counter();
b(); //0
b(); //1
b(); //2
b(); //3
